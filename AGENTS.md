# QuestLunch — Project Context

**Who this is for:** solo founder-led project, currently pre-build, no existing users or codebase. Design/market strategy already done (see the accompanying strategy doc) — this is the build phase.

**What the product is:** QuestLunch (working name) turns picking lunch into a short game — pick a craving, get routed to a matched restaurant on a live map, and if the queue's too long, branch into "Roam Mode" for nearby alternatives that match the same craving.

**Why it exists / the wedge:** Beli proved restaurant discovery can go viral through social gamification (75M+ rankings, out-grew Yelp), but it's asynchronous — ranking where you've *already* eaten. Waitlist apps (Waitwhile, ScanQueue, WaitTime) solve the live-queue problem but are pure utility with no discovery or sharing layer. Nobody combines live-queue routing + social gamification + a referral-funded channel for low-traffic restaurants. That's the gap.

**Current priority: virality, not monetization.** No ads, no paywalls, no revenue features in v1. Every feature decision should be judged on whether it drives sharing, referrals, or return visits — not revenue. Rewards exist to fuel word-of-mouth.

**The other business goal baked into the product:** driving foot traffic to *less-trafficked* restaurants. This isn't charity — it's the acquisition channel: restaurants that opt in to a "Boost Slot" get preferential placement in Roam Mode and only pay out a reward (discount/free item) when a quest actually converts into a visit. That gives the app inventory of good quests to offer, and gives restaurants a pay-for-results channel instead of upfront ad spend.

**Platform call already made:** build web first, not native app. A shareable quest link opening instantly in a browser (no app-store install friction, no review wait) is the better vehicle for a viral loop than a link that dead-ends at a store listing. Native app comes later, once there's a retained cohort worth the geofencing/push investment.

**Tone/visual direction:** should feel like a game (Duolingo/BeReal energy), not a utility app — food-warm color palette (terracotta, mango, matcha), playful motion, a mascot. The onboarding needs a real "wow" moment in the first few seconds, not a feature-bullet carousel.
